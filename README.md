# Hotel App - Приложение для просмотра номеров отеля

## Описание
Мобильное приложение для Android, позволяющее просматривать и искать номера в отеле.

## Функциональность

### Авторизация и регистрация
- **Окно авторизации**: вход в систему по логину и паролю
- **Окно регистрации**: создание новой учетной записи с указанием ФИО

### Главное меню
- Приветствие с отображением ФИО пользователя
- Три кнопки:
  - Поиск номеров по фильтрам
  - Просмотр всех номеров
  - Выход из учетной записи

### Поиск номеров
- **Фильтры поиска**:
  - Дата заезда (DatePicker)
  - Дата выезда (DatePicker)
  - Количество гостей
- Отображение только доступных номеров по заданным критериям

### Просмотр всех номеров
- Список всех номеров с фильтрацией:
  - Все номера
  - Доступные номера
  - Недоступные номера
- Клик по номеру открывает детальную информацию

### Детали номера
- Номер комнаты
- Тип номера
- Вместимость
- Цена за ночь
- Описание
- **Удобства** (новое!):
  - Wi-Fi
  - Телевизор
  - Кондиционер
  - Минибар
  - Балкон
- Статус доступности
- Фотография (заглушка)

## База данных SQLite

### Таблицы:
1. **users** - пользователи системы
2. **rooms** - номера отеля (10 тестовых номеров с удобствами)
3. **bookings** - бронирования

### Поля таблицы `rooms`:
- id, room_number, room_type, capacity, price_per_night, description, image_path
- **has_wifi** (boolean) - наличие Wi-Fi
- **has_tv** (boolean) - наличие телевизора
- **has_air_conditioning** (boolean) - наличие кондиционера
- **has_minibar** (boolean) - наличие минибара
- **has_balcony** (boolean) - наличие балкона

### Тестовые пользователи:
- Логин: `1`, Пароль: `1` (Иванов Иван Иванович)
- Логин: `2`, Пароль: `2` (Петров Петр Петрович)

### Тестовые номера:
- 10 номеров различных типов:
  - Эконом (1 человек) - 2000 ₽/ночь
  - Стандарт (2 человека) - 3000 ₽/ночь
  - Делюкс (2 человека) - 4500 ₽/ночь
  - Люкс (3 человека) - 5500 ₽/ночь
  - Семейный (4 человека) - 6000 ₽/ночь
  - Президентский (5 человек) - 10000 ₽/ночь

Некоторые номера имеют тестовые бронирования для демонстрации статусов "Доступен"/"Занят".

## Установка и запуск

### Предварительные требования:
- Установите Android Studio (рекомендуется последняя стабильная версия)
- Убедитесь, что установлен Android SDK (API 33)
- JDK 17 или выше

### Шаги установки:

1. **Откройте проект в Android Studio**
   - File → Open → выберите папку HotelApp

2. **Настройте local.properties**
   - Откройте файл `local.properties`
   - Убедитесь, что путь к SDK правильный:
     ```
     sdk.dir=C\:\\Users\\ВАШЕИМЯ\\AppData\\Local\\Android\\Sdk
     ```
   - Если файл не создается автоматически, создайте его вручную

3. **Дождитесь синхронизации Gradle**
   - Android Studio автоматически начнет скачивать зависимости
   - При ошибках выберите File → Sync Project with Gradle Files

4. **Подключите устройство или запустите эмулятор**
   - Для эмулятора: Tools → AVD Manager → Create Virtual Device
   - Рекомендуется API 33 (Android 13)

5. **Запустите приложение**
   - Нажмите Run (зеленая кнопка Play) или Shift+F10

### Устранение проблем:

**Ошибка: SDK location not found**
- Откройте файл `local.properties` и добавьте путь к SDK

**Ошибка: Unsupported class file major version**
- Убедитесь, что используете JDK 17
- File → Project Structure → SDK Location → выберите JDK 17

**Ошибка: Failed to resolve dependencies**
- File → Invalidate Caches / Restart
- Удалите папку `.gradle` и `.idea` в корне проекта
- Повторите синхронизацию

## Использование

1. **Вход**: используйте тестовые данные (логин: `1`, пароль: `1`)
2. **Главное меню**: выберите нужное действие
3. **Поиск**: укажите даты и количество гостей для поиска доступных номеров
4. **Просмотр**: просматривайте все номера с фильтрацией по доступности
5. **Детали**: кликните на номер для просмотра подробной информации

## Добавление изображений номеров

Для добавления реальных фотографий номеров:

1. Поместите изображения в папку `app/src/main/res/drawable/`
2. Именуйте файлы как `room_101.jpg`, `room_102.jpg` и т.д.
3. Обновите код в `DatabaseHelper.kt` при создании номеров, указав названия файлов вместо "placeholder"
4. Обновите `RoomAdapter.kt` и `RoomDetailsActivity.kt` для загрузки реальных изображений:
   ```kotlin
   val resourceId = itemView.context.resources.getIdentifier(
       room.imagePath, "drawable", itemView.context.packageName
   )
   if (resourceId != 0) {
       ivRoomImage.setImageResource(resourceId)
   }
   ```
